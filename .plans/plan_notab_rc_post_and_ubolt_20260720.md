# RC 기둥 x 계측 + 세로 치수 정합 + U-bolt 태그 (cycle 91)

## 1. 배경 — cycle 90에서 드러난 사실

`member-anchor source=member-geometry ... handle=7A box=(285.5,258.9)~(385.5,378.9)`

세로 MEMBER로 `7A`를 골랐으나 `7A`는 **100×120 큰 박스**다. `7C`는 페이퍼 `17.82×17.82`
**정사각형**이라 세로성 신호가 없다.

> **결론: 세로 기둥은 독립 솔리드가 아니다.** `7A`가 기둥+가로재+베이스 플레이트를 통째로
> 덮고 있고, `7C`는 볼트/너트류다. 450을 기하에서 못 뽑았던 것과 **같은 상황**이다.

분류 규칙을 다듬어도 없는 것은 찾을 수 없다. 앵커가 `7A` 중앙(335.5, 318.9)에 찍혀
기둥과 가로재 사이 빈 공간을 가리킨 것이 현재 증상이다.

## 2. 이번 사이클 = 계측 우선. 앵커 확정은 다음 사이클.

비율 추정을 또 얹지 않는다. **기둥의 실제 x를 먼저 측정**하고, 그 값으로 다음 사이클에
파라미터 기반 앵커를 확정한다.

## 3. 항목

### ① 기둥 x 위치 계측
알고 있는 값:
- 세로 범위 = `minY` ~ `minY + F2 × vScale` (F2는 정상 동작 중: RC2 500, RC3 600)
- 기둥 두께 ≈ 프로파일 높이 65 × scale
- 총 폭 = `A + A1`, 분할 = `A2`(없으면 `A1`)

**모르는 것 = 기둥의 x**. RC2에서 베이스 플레이트가 부재보다 40 넓으므로
기둥이 `minX`에 붙어 있는지, 안쪽으로 들어와 있는지 확정되지 않았다.

**★원천 발견 (2026-07-20): 서포트 포트(`PPorts`)가 답이다.**

`HANTEC.RC1()`은 앵커를 bbox가 아니라 **포트**에서 뽑는다.
```csharp
// HANTEC.cs:868~888
Point3d point1 = this.ConvertPortToPoint(this.PPorts[2]);   // F1
Point3d point2 = this.ConvertPortToPoint(this.PPorts[1]);   // F2 ← 세로 부재
Point3d point3 = this.ConvertPortToPoint(this.PPorts[3]);   // P1
```
포트는 **병합 솔리드와 무관하게 부재별 부착점을 그대로 보유**한다.
bbox 역산도, 기하 분리도 불필요하다.

계측 방법(**작도 변경 없이 로그만**):
- 원본 DB에서 서포트의 **포트 전량을 덤프**한다(인덱스·이름·WCS 좌표).
  메모리 `p5-topology-identification`에 따르면 `Part.GetPorts` 경로가 존재한다.
- 각 포트를 `NotabProjectWcsToPaper()`로 투영해 **페이퍼 좌표**를 함께 남긴다.
- RC1/RC2/RC3에서 `PPorts[1]`(F2)이 실제로 기둥 위에 오는지 대조한다.

폐기된 방법(자문 결과):
- (a) `7A` bbox ↔ `A/A1/A2` 역산 — **순환 논증 위험**. 같은 union solid의 외곽을
  재표현할 뿐이라 독립 관측값이 아니다(Codex 지적).
- (b) 2D 선분 히스토그램 — **불가능**. 상세도는 실제 2D 엔티티가 아니라
  복제 `Solid3d` + 뷰포트 와이어프레임이라 읽을 `Line`이 없다(Codex 확인).

**이번 사이클은 덤프·대조까지. 추정 앵커를 넣지 않는다.**

### ② 세로 치수 기준 정합
`verticalX = minX - offset - dimClear`인데 `minX`는 **플레이트 기준(490 폭)**이다.
가로 치수만 부재 기준(450)으로 옮겨 서로 어긋난다(RC2에서 500 치수가 안 움직인 원인).
- 세로 치수의 기준 x를 **가로 치수와 같은 부재 구간**으로 맞춘다.
- 가로가 legacy 폴백이면 세로도 legacy 기준을 쓴다(기준 혼용 금지).
- 사용된 기준을 로그에 남긴다.

### ③ U-bolt 태그 덤프 (조사)
- 추출 파이프라인은 `TryStripCleanSolidMetadata()`로 XData·확장사전을 제거하므로
  **상세도 솔리드에서는 태그를 읽을 수 없다.**
- 따라서 **원본 DB에서 캡처**해야 한다. `auto-include`가 U-bolt를 별도 객체로 잡는 정황
  (`addedSupport=1`)이 있으므로, 그 시점에 **속성 전량을 로그로 덤프**한다.
- **이번 사이클은 조사·덤프까지.** 도면 표기는 다음 사이클.
