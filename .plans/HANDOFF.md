## 왜 이 사이클인가
밸룬 3개가 앵커에서 멀리(r=129~149) 떨어진 한 곳으로 몰린다.
**같은 문제에 3회 실패**했다: ①앵커 중앙 이동(r 급증) ②서포트 리더 면제(수치 무변화)
③패딩 분리 + 최소이격 축소(개선 없음, 일부 악화).
**상수 튜닝을 중단하고 전용 경로로 분리한다.**

## 확정된 원인 (자문 검증, 재조사 금지)
`costRefExt`는 밸룬별 앵커가 아니라 `AppendNotabPaperDimensions()`에서 **한 번** 설정된다
(서포트 + 파이프 중심 bbox). **밸룬 앵커는 비용 기준에 포함되지 않는다.**
`preferDown=false`, `angleW=0`이므로 실질 비용은
```
cost = costRefExt 밖으로 나간 양 + radius × 0.01
```
→ **거리가 사실상 공짜**다. r이 100 늘어도 비용은 1. 기준상자 이탈은 그대로 가산.
RC2 F1 `r=129.3, cost=48.79` = 반경항 1.29 + 이탈 47.5.

**★거리 가중치만 올리는 것은 네 번째 튜닝 실패가 될 가능성이 높다**(자문 명시).
`TryPlace`는 텍스트 콜아웃 전제를 다섯 개 갖는다: 좌우 한쪽 강제 / `minDx` 강제 /
외접 사각형 충돌 / 앵커→꺾임→상자 2선분 리더 모델 / 서포트 bbox 안쪽 선호.

## A. 밸룬 전용 경량 배치 신설
`TryPlace`를 밸룬에 쓰지 않는다. 별도 탐색을 만든다.
```
후보 중심 = anchor + 방향벡터 × 거리     // 작은 거리부터 확장, 첫 허용 구간에서 최저 비용 채택
비용 = 거리 × 높은 가중치
     + 원·보조문자 bbox 겹침(하드 제약 또는 큰 페널티)
     + 리더 교차 페널티
     + 도면 밖 페널티
costRefExt 이탈 항은 밸룬에서 제거하거나 아주 약하게
```
- **360° 방향 탐색**. 좌우 강제는 **소프트 선호**로 강등.
- `minDx`는 **제거 또는 반지름 수준**으로 축소.
- **텍스트 콜아웃의 R1~R4·minDx·비용식은 그대로 보존**(회귀 금지).

## B. 앵커 = 중심 유지, 리더는 외곽 접점에서 출발
```
의미 앵커   = 부재 중심(현행 vertical-mid / horizontal-mid 유지)
리더 시작점 = 밸룬 방향을 향한 부재 외곽 접점
```
지시 의미는 유지하면서 리더가 부재 내부·치수 통로를 길게 가로지르지 않게 한다.
기존 `BoxEdgeToward()`가 유사 계산을 갖고 있으니 재사용을 검토할 것.

## C. 치수 리더 면제 — 범위를 좁게
**밸룬 전용 / 리더 선분 검사만 / 치수 장애물만** 면제.
원·보조문자 **겹침 금지는 유지**.
- 위험: 치수선·보조선·문자 위를 리더가 지나 판독성이 떨어질 수 있다.
- 현재는 치수 전체 `GeometricExtents`를 사각형 하나로 다뤄
  "문자는 피하고 선만 통과"를 구분하지 못한다. **이번엔 구분하지 말고 범위만 좁게 적용**하고,
  판독성 문제가 실제로 보이면 다음 사이클에서 문자/선분 영역을 분리한다.

## D. 진단
`balloon-draw`에 **`tier`를 반드시 포함**(tier 2는 리더 검사가 꺼진 상태다).
거리·방향·거절 사유별 카운트를 남겨 실패 시 원인이 즉시 갈리게 할 것.

## E. 주의 (자문 지적)
- 밸룬을 **외접 사각형**으로 충돌 판정하면 원의 빈 모서리까지 점유로 봐 후보가 과다 탈락한다.
  원형 판정 또는 축소된 사각형을 검토.
- `calloutLeader=974`가 최대 거절원이다. **밸룬을 마지막에 배치하는 정책의 필연적 결과**이므로
  짧은 리더 우선 정책이 반드시 필요하다.
- `TryPlace`의 반환 `center`는 실제 원 중심이 아니라 **텍스트 상자 경계 중앙**이다.
  전용 배치기로 분리하지 않으면 유지보수 시 재오해 위험이 크다.
- 번호(`F1`) MText 실제 폭이 반지름보다 넓을 수 있다. **최소 반지름을 문자 폭으로 검증**할 것.
