# 무탭 콜아웃 배치 — 근본 진단 원장 (2026-07-19)

> 3-스트라이크(배치 이슈 4회 반복 cycle72/75/76/77/78) 도달 → 재빌드 중단, 코드 정독 진단. 집도 전 단계.

## 증상 (라이브)
- GD1: 부재 콜아웃 글자 방향 부자연.
- GD2: 라인넘버 콜아웃이 부재에 간섭, C 부재 콜아웃 방향·부재 간섭.
- GD3: 앵글 콜아웃 화살표가 앵글 중심이 아니라 파이프에 걸침(어중간). 복합 간섭.

## 확정 근본 원인 (코드 정독)
1. **[실버그] 리더-부재 충돌검사 무력화** — `SegIntersectsBox`([Commands.cs:103](../PlantFlow_Support/Core/Commands.cs#L103))가 `if (PointInBox(start, box)) return false;`. 파이프/부재 앵커는 서포트 bbox **안**이라 리더-서포트 검사가 **항상 통과** → 리더가 부재를 뚫어도 안 잡힘.
2. **[구조 한계] 통짜 bbox 장애물** — 서포트를 `supportPaperExt` 1개 사각형으로 모델(4416 `TryComputeNotabDimPaperGeometry`: supportExt 8코너만 투영). 얇은 부재+빈틈(파이프 위치)을 구분 못 함. #1만 고치면 내부앵커 리더가 전부 bbox 교차→relaxed 추락→무의미.
3. **[부수] 최근접 모서리 계산이 앵커 bbox 밖이면 붕괴** — GD2 파이프 앵커 X=415.5 > maxX=404 → dR 음수 → 방향 오판 → tier=1.
4. **[부수] 텍스트 여백 없음** — bbox 바로 밖(≈1.6) 배치라 부재에 닿아 보임.
5. **[앵커 정확도] 부재 실중심 미지정** — gd3Two 등 앵커가 `minX+width*0.25` 비율 추정 → GD3 앵글 실중심 아닌 파이프에 걸침.

## 사용자 확정 스펙 (2026-07-19)
- **장애물 = 활성 서포트(예 GD1-001)의 자기 부재 바만**. 문자·치수·리더가 자기 부재와 간섭 금지.
- **파이프·U볼트·타 객체 = 장애물 아님**. 리더가 뚫고 가도 허용(접촉 판정 대상 아님).
- 부재 콜아웃 화살표는 **부재 실중심**을 가리켜야 함(GD3 앵글).

## 해결 아키텍처 (통짜 bbox → 부재 바 개별)
활성 서포트의 **부재 바 2D 페이퍼 박스**를 뽑아 (a)장애물 (b)앵커 실중심 (c)파이프/U볼트 제외를 동시 해결.

### 실현성 (확인됨)
- **투영 프리미티브 존재**: `NotabProjectWcsToPaper(vp, wcs)`([Commands.cs:4470](../PlantFlow_Support/Core/Commands.cs#L4470)) — 모델 WCS→페이퍼.
- **부재 개별 solid 존재**: 파이프라인이 `ExplodeModelSpaceToSolidIds`로 서포트를 Solid3d 리스트로 분해(2383). 평면화 후 디테일 DB model space에 존재.
- **콜아웃 시점 접근 가능**: 콜아웃/치수 스테이지가 디테일 `db`+`vp` 보유(4519-4532). 그 model space의 Solid3d를 열거→각 `GeometricExtents`(WCS)→코너 투영→부재별 페이퍼 박스 산출 가능.

### 남은 설계 결정 (집도 전 확정 필요)
- **Q1 부재 vs 파이프/U볼트 분리**: 디테일 model space 열거 시 파이프(원형 단면, `s_isoPipeCenterWcs` 근처)·U볼트를 제외하는 판정 기준. (후보: 단면형상/위치/소스태그)
- **Q2 부재→designation 매핑**: 어느 solid가 "앵글(L)"인지 지정해 앵커를 그 부재 실중심에 꽂기. (후보: 단면 프로파일 대조 angle vs channel)
- **Q3 앵커 규칙**: 부재 실중심 = 해당 부재 페이퍼 박스 중심으로 통일(타입별 비율 하드코딩 제거).

### 집도 계획 골격 (확정 후)
1. `TryComputeNotabMemberBoxes(vp, db) → List<(box, designation?)>` 신설(부재별 페이퍼 박스, 파이프/U볼트 제외).
2. placer 장애물 = 부재 박스들(통짜 bbox 대체). `SegIntersectsBox` 내부-시작 버그 교정(부재 바 기준이면 앵커가 자기 바 위라도 타 바 교차는 검출).
3. 앵커 = 대상 부재 박스 중심(Q2 매핑). gd2Two/gd3Two 비율 앵커 폐기.

## 설계 결정 확정 (사용자 2026-07-19)
- **Q1 = 단면형상으로 분리**: 2D 페이퍼에서 부재=사각형, 파이프=원/타원, U볼트=가는 선/호. 원·가는선은 장애물 제외.
- **Q2 = solid→designation 매핑**: 부재 사각형의 치수(높이/폭)를 BOM 프로파일(예 L-75×75×9→75, C-150→150)과 대조해 앵글/채널 식별. 콜아웃 앵커를 해당 부재에 꽂음.
- **Q3 = 앵커 = 부재 박스 중심 통일 (동의)**. gd2Two/gd3Two 비율 앵커 전면 폐기.
- 추가 결함: GD3 전 콜아웃 방향 오류(빽빽 → 대각 다 막혀 수직/수평 추락, bbox 근본). 수직 리더(textX==anchorX)는 좌우 규칙 붕괴 → **대각 강제 필요**.
- 미해결 의심: 렌더≠로그(GD1 L 로그=우측인데 도면=좌측) → 실제 그려진 MText extents 측정으로 확정 필요.
