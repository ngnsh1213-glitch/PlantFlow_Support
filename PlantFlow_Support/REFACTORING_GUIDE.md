# PlantFlow Support 리팩토링 가이드

## 개요 (Overview)
이 문서는 코드의 유지보수성과 구조를 개선하기 위해 수행된 리팩토링 및 폴더 구조 변경 사항을 다룹니다. 기존의 거대한 `PSUtil` 및 `PlantOrthoView` 클래스를 기능별로 분해하고, 파일들을 논리적인 폴더로 재배치하였습니다.

## 폴더 구조 (Folder Structure)

### 1. `Utils/` (유틸리티 클래스)
기존의 비대한 `PSUtil` 역할을 대체하는 전문화된 정적(Static) 헬퍼 클래스들을 포함합니다.
*   **`GeomUtils.cs`**: 기하학 및 수학 계산 (벡터, UpVector, 회전 로직, 미터/임페리얼 변환 등).
*   **`AnnotationUtils.cs`**: AutoCAD 치수(Dimension), 지시선(MLeader), Quarter Block 및 텍스트 주석 생성 로직.
*   **`DatabaseUtils.cs`**: SQLite 데이터베이스 헬퍼 (`PnPDatabase`, `DataTable` 처리 및 SQL 쿼리).
*   **`CadUtils.cs`**: AutoCAD 상호작용 헬퍼 (Zoom, 문서 잠금, 트랜잭션 관리).
*   **`ProjectDataUtils.cs`**: 파일 I/O 및 JSON 설정 처리.
*   **`UIUtils.cs`**: 일반적인 UI 유틸리티 및 헬퍼 메서드 (루트 폴더에서 이동됨).
*   **`LocalResources.cs`**: 문자열 리소스 및 지역화 헬퍼.

### 2. `Ortho/` (Ortho 생성 로직)
Ortho 뷰 생성 및 처리를 위한 핵심 로직을 포함합니다.
*   **`PlantOrthoView.cs`**: Ortho 처리의 메인 컨트롤러/진입점. 전체 프로세스를 조율하지만, 구체적인 작업은 매니저 클래스에 위임합니다.
*   **`OrthoViewportManager.cs`**: **[신규 클래스]** 뷰포트 생성, 스케일 조정, `ProccessCurrentViewport` 로직, BOM 및 주석 배치를 전담하는 매니저입니다.
*   **`HANTEC.cs`**: 특정 설계 표준(HANTEC) 구현 로직.
*   **`PlantAutoCoding.cs`**: 서포트 자동 코딩/태깅 로직.

### 3. `UI/` (사용자 인터페이스)
Windows Forms 및 Palette 컨트롤을 포함합니다.
*   **`PaletteTab.cs` / `.Designer.cs` / `.resx`**: 애플리케이션의 메인 도킹 팔레트 UI.
*   **`formDBSetting.cs`**: 데이터베이스 설정 폼.
*   **`formExistedStdType.cs`**: 기존 표준 타입 선택 다이얼로그.
*   **`FormProjectScanResult.cs`**: 스캔 결과 표시 다이얼로그.
*   **`CustomPaletteSet.cs`**: AutoCAD PaletteSet 컨테이너 클래스.
*   **`UISettings.cs`**: UI 설정 저장소.

### 4. `Models/` (데이터 모델)
데이터 구조 및 엔티티를 포함합니다.
*   **`SupportInfo.cs`**: 선택된 서포트의 속성 정보(이름, ID, 범위 등)를 담는 핵심 데이터 클래스.
*   **`AttachmentInfo.cs`**: 부착된 부품(Pad 등)에 대한 정보 클래스.
*   **`BOMs.cs`**: 물량 산출(BOM) 데이터 구조 및 생성 로직.
*   **`PlantSupportName.cs`**: 서포트 이름 파싱 헬퍼.

### 5. `Core/` (애플리케이션 코어)
진입점 및 명렁어 정의를 포함합니다.
*   **`Commands.cs`**: AutoCAD 명령어 정의 (예: `PFS`, `Export2D`, `OrthoView`).
*   **`GlobalUsings.cs`**: 프로젝트 전역 네임스페이스 지시문.

## 주요 변경 사항 (Key Changes)
*   **PSUtil Facade 패턴**: `PSUtil.cs`는 루트 폴더에 유지됩니다. 이 클래스는 이제 "껍데기(Facade)" 역할만 수행하며, 실제 작업은 새로운 `Utils/*` 클래스들에게 위임합니다. 이를 통해 `PSUtil`을 호출하는 기존 코드와의 호환성을 유지했습니다.
*   **OrthoViewportManager 도입**: 뷰포트 스케일링, 레이어 관리, 주석 배치와 관련된 복잡한 로직을 `PlantOrthoView.cs`에서 분리하여 `OrthoViewportManager.cs`로 이관했습니다. 이를 통해 스케일 리셋 문제 및 컨텍스트 손실 버그를 해결했습니다.
*   **네임스페이스 및 리소스 경로**: 파일 이동에 따라 `.csproj` 파일이 업데이트되었으며, Embedded Resource(폼 리소스 등) 경로가 `UI/` 폴더를 가리키도록 수정되었습니다.
